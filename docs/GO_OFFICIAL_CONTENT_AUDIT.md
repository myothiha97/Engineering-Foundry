# Go Runtime Lab: official content audit

Verified on 2026-07-16 against current official Go documentation. This matrix covers the 40 active lessons shown in the guided curriculum. It is a maintenance checklist: when a lesson changes, its learning objectives and core rules should still match the linked source.

The lab intentionally rewrites the material in a TypeScript-developer-friendly voice. The official pages remain the authority for language and library behavior. The curriculum covers the core language and the practical standard-library path a new Go developer needs; it does not try to reproduce every specification edge case or every standard-library package.

## Gaps corrected during this audit

- Explained that Go builds packages; it does not choose a source file to execute first. `main.go` is a filename convention, while `package main` and `func main` have language meaning.
- Added inferred declarations (`var name = "myo"`, `name := "myo"`), the complete basic type families, aliases, and `iota`.
- Added current integer/iterator range forms and the `clear` built-in for slices and maps.
- Split time from context so context appears after channels, then restored important time coverage: RFC3339, reference layouts, UTC, and locations.
- Corrected generic JSON numbers: `float64` is the default, while `Decoder.UseNumber` preserves `json.Number`.
- Added current `WaitGroup.Go` guidance, `time.LoadLocation`, and reflection's `CanSet` guard.
- Added strict JSON request decoding, HTTP response-status checks, server timeouts and body limits, test cleanup helpers, and file-close error guidance for writes.
- Replaced the overloaded startup pipeline with the official five-step build/run model and made the toolchain-versus-language guarantee boundary explicit.
- Added an automated paragraph-density limit and official-reference requirement for every active Go lesson.
- Kept best practices visible in chapter 03 for every lesson without adding another navigation stage.
