# Implementation roadmap

## Delivered vertical foundation

- Monorepo, strict TypeScript, shared UI, diagrams, editor, schema, engine, database, authentication boundary, analytics contract, Docker, CI, and docs.
- Go Module 0 lesson: source → compiler → linker → loader → runtime → initialization, with complete stage structure and execution-inspector project definition.
- Backend Module 0 lesson: process resources, sockets, signals, readiness, draining, and termination, with architecture evolution, request-timeline experiment, failure simulator, trade-off lab, and LedgerFlow Stage 1 definition.
- Anonymous notes, bookmarks, progress, focus, predictions, deterministic code validation, and evidence-based mastery claim.

## Next: foundation hardening

1. Finish account screens, email verification, recovery, rate limiting, and authorization tests.
2. Implement idempotent anonymous-progress migration and server mutations.
3. Add dashboard, global search index, concept graph, and review queue UI.
4. Add component-test coverage and visual-regression baselines.
5. Generate a complete Drizzle migration from the schema and test upgrade/rollback on PostgreSQL.

## Then: content expansion

Expand Go Module 0 into focused lessons and build Module 1 only after learning evidence validates the format. Expand Backend Module 0 similarly, then build Networking & HTTP with the full 18-stage request explorer. Do not bulk-generate the remaining curriculum.

## Later

Secure Go runner service, hidden server-side tests, observability, production deployment, accessibility audit, load testing, and a formal security review.
